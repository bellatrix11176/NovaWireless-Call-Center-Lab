# NovaWireless Call Center Lab

**Research codebase for:** Aulabaugh, G. (2026). *When KPIs Lie: Governance Signals for AI-Optimized Call Centers.*

---

## What This Is

This repository contains the fully synthetic simulation environment used to generate the findings in the paper above. It produces 12 months of call center activity — customer interactions, agent behavior, and outcome data — across 11 scenario types that span clean resolutions, metric gaming, account fraud, and loyalty offer failures.

The environment is designed to answer a question that real call center data cannot: **what is the gap between what gets recorded in the CRM and what actually happened to the customer?** In real operations, ground-truth resolution is unobservable. Here, it is measured precisely — which is how the 42 percentage-point Goodhart gap in the paper was documented.

---

## The Core Finding

> Proxy FCR held near **89%** all year. True durable FCR was approximately **47%**. The dashboard said everything was fine. Q4 churn exceeded forecast by **11.8 percentage points** and triggered emergency staffing response.

The gap was not caused by bad agents. It was caused by a measurement system that rewarded the appearance of resolution over actual resolution — and rational agents responded to the incentives they were given.

## Getting the Data

You have two options:

**Option A — Download the pre-generated dataset (recommended):**

All large data files are available in the [v1.0 Release](https://github.com/bellatrix11176/NovaWireless-Call-Center-Lab/releases/tag/v1.0). Download the files you need and place them in the folders shown in the release description. This gives you the exact data used in the paper.

**Option B — Regenerate everything from scratch:**

Run the full pipeline and it generates all data files automatically:

```bash
pip install pandas numpy matplotlib
python run_all.py
```

The `output/` folder is empty in the repo by design — it gets populated when you run the pipeline. All figures from the paper are regenerated by running:

```bash
python novawireless-call-gen/src/pressure_experiment.py
```

## The 11 Scenarios

| Scenario | What Happens | Proxy FCR | True FCR | Gap |
|----------|-------------|-----------|----------|-----|
| `clean` | Issue genuinely resolved | 0.90 | 0.92 | −0.02 |
| `unresolvable_clean` | Issue has no solution; rep is honest | 0.55 | 0.10 | +0.45 |
| `gamed_metric` | Rep closes call with unauthorized credit to protect FCR | 0.91 | 0.18 | +0.73 |
| `fraud_store_promo` | Store rep applies unauthorized promo post-call | 0.88 | 0.25 | +0.63 |
| `fraud_line_add` | Line added without customer consent | 0.85 | 0.22 | +0.63 |
| `fraud_hic_exchange` | HIC device exchange processed fraudulently | 0.87 | 0.15 | +0.72 |
| `fraud_care_promo` | Care rep applies unauthorized promotional credit | 0.92 | 0.30 | +0.62 |
| `activation_clean` | New line activation completed successfully | 0.93 | 0.95 | −0.02 |
| `activation_failed` | Activation fails; rep marks resolved anyway | 0.45 | 0.08 | +0.37 |
| `line_add_legitimate` | Legitimate line addition | 0.90 | 0.92 | −0.02 |
| `loyalty_offer_missed` | Customer shopping competitors; rep applies band-aid discount instead of trade-in offer | **0.93** | **0.14** | **+0.79** |

The `loyalty_offer_missed` scenario drives the Q4 collapse. It has the highest proxy FCR of any gaming scenario — the rep appears to resolve the call — but the lowest true FCR and the highest churn multiplier (1.80×), because the competitor offer that triggered the call is never addressed.

## The Five Governance Signals

The paper introduces five signals for detecting Goodhart divergence before it reaches financial visibility:

| Signal | Full Name | What It Detects |
|--------|-----------|----------------|
| **DAR** | Delayed Adverse Rate | Adverse outcomes 31–60 days after a "resolved" call — the signature of band-aid credits expiring |
| **DOV** | Durable Outcome Validation | Correlation between CRM disposition and true 60-day outcome — measures how honest the recorded resolution is |
| **DRL** | Downstream Remediation Load | Volume of future work (escalations, credits, churn processing) traceable to previously "resolved" calls |
| **POR** | Proxy Overfit Ratio | Rate of proxy FCR improvement relative to true FCR improvement — POR > 2.0 signals active gaming |
| **TER** | Terminal Exit Rate | Share of customers who churn within 90 days of a credit-based resolution — measures whether credits are retaining customers or just delaying churn |

## Pressure Experiment

`pressure_experiment.py` runs a controlled comparison of baseline vs. high-pressure workforce conditions and a 12-month time series showing the Q4 collapse. It produces five figures:

- **Figure 1** — Scenario mix shift under pressure
- **Figure 2** — KPI comparison (True FCR, Proxy FCR, Repeat 30d, Repeat 31-60d, Churn)
- **Figure 3** — Rep burnout and strain distribution
- **Figure 4** — Per-scenario Goodhart gap
- **Figure 5** — Three-panel Q4 breaking point (FCR gap / repeat contacts / churn vs. forecast)

Output is written to `output/call-gen/experiments/`.
